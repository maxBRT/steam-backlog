# Secrets Quick Reference

Quick reference for managing secrets in Steam Backlog.

## Local Development Setup

```bash
# 1. Copy environment template
cp .env.example .env

# 2. Generate application key
php artisan key:generate

# 3. Get Steam API key
# Visit: https://steamcommunity.com/dev/apikey
# Register domain: localhost
# Copy key and add to .env:
STEAM_API_KEY=your_key_here
```

## Required Secrets

| Secret | Where to Get | Required For |
|--------|--------------|--------------|
| `APP_KEY` | Auto-generated by `php artisan key:generate` | Encryption, sessions |
| `STEAM_API_KEY` | [steamcommunity.com/dev/apikey](https://steamcommunity.com/dev/apikey) | Library sync (GetOwnedGames) |
| `DB_PASSWORD` | Auto-configured on Laravel Cloud | Database access (prod only) |

## Environment Variables Reference

### Essential

```bash
APP_KEY=                    # Laravel encryption key
APP_ENV=local               # local | production
APP_DEBUG=true              # false in production
APP_URL=http://localhost    # Your app URL

STEAM_API_KEY=              # From Steam Web API
STEAM_RETURN_URL="${APP_URL}/auth/steam/callback"
# STEAM_REALM is optional (host only). Leave unset to derive from STEAM_RETURN_URL.
```

### Database (Local)

```bash
DB_CONNECTION=sqlite
```

### Database (Production)

```bash
DB_CONNECTION=pgsql
DB_HOST=                    # From Laravel Cloud
DB_PORT=5432
DB_DATABASE=steam_backlog
DB_USERNAME=                # From Laravel Cloud
DB_PASSWORD=                # From Laravel Cloud
```

### Sessions

```bash
SESSION_DRIVER=database
SESSION_LIFETIME=10080              # 7 days in minutes
SESSION_SECURE_COOKIE=              # null locally, true in prod
SESSION_HTTP_ONLY=true
SESSION_SAME_SITE=lax
```

## Security Rules

### ✅ DO

- Use `.env` for local secrets
- Use environment variables for production (Laravel Cloud)
- Keep different keys per environment
- Rotate secrets periodically
- Add new secrets to `.env.example` (without values)
- Document new secrets in `docs/secrets-management.md`

### ❌ DON'T

- Commit `.env` files to git
- Hardcode secrets in application code
- Share production keys in Slack/email
- Reuse `APP_KEY` across environments
- Use weak or simple secret values
- Log secret values

## Git Safety

Files already in `.gitignore`:
```
.env
.env.backup
.env.production
.env.local
.env.*.local
```

Check before committing:
```bash
git status | grep -i env
```

## Common Issues

### "No application encryption key"
```bash
php artisan key:generate
```

### Steam API returns 403
- Check `STEAM_API_KEY` is set in `.env`
- Verify key at [steamcommunity.com/dev/apikey](https://steamcommunity.com/dev/apikey)
- Confirm domain registration matches (localhost vs production)

### Sessions not working
- Verify `SESSION_DRIVER=database`
- Run `php artisan migrate`
- Check `sessions` table exists

## Production Deployment

Before deploying to Laravel Cloud:

1. **Set environment variables in Laravel Cloud dashboard:**
   - `APP_KEY` (generate with `php artisan key:generate --show`)
   - `STEAM_API_KEY` (production key, registered with prod domain)
   - `APP_ENV=production`
   - `APP_DEBUG=false`
   - `SESSION_SECURE_COOKIE=true`

2. **Verify configuration:**
   - Database credentials (auto-configured by Laravel Cloud)
   - `APP_URL` matches production domain
   - `STEAM_RETURN_URL` uses production URL

3. **Run checklist:**
   - See `docs/security-checklist.md`
